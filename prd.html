1. Executive Summary

Sprout Support Agent is a Chrome sidebar extension (Chrome Side Panel API) built for L4 support engineers. It surfaces real-time EAB (Escalation, Advocacy, Bypass) metrics from the Customer Advocacy Tool (CA Tool), provides AI-powered ticket investigation using Azure OpenAI with RAG, and delivers instant access to process FAQs and quick tools â€” all without leaving the Jira ticket context.

The extension addresses critical pain points in L4 support workflows: fragmented tools, manual investigation overhead, and lack of intelligent assistance. By embedding support intelligence directly in the browser sidebar alongside Jira, Sprout reduces investigation time, increases workaround adoption, and empowers engineers with AI-driven insights.

2. Problem Statement

L4 support engineers face significant friction when resolving complex customer escalations. The following pain points drive the need for Sprout Support Agent:

Fragmented Tool Landscape

Engineers must context-switch between Jira, the CA Tool (Customer Advocacy Tool), internal wikis, and Slack â€” losing focus and time.

Manual Investigation Overhead

Finding related tickets, root causes, and historical workarounds requires manual search across multiple systems with no intelligent summarization.

Process Knowledge Gaps

New hires and even seasoned engineers struggle to locate up-to-date process documentation and escalation paths quickly.

Reactive Metrics Visibility

EAB metrics (Escalation, Advocacy, Bypass) are not readily available in-context when triaging tickets, leading to delayed awareness.

Inconsistent Workaround Adoption

Proven workarounds exist in historical tickets but are not surfaced at the right moment, resulting in redundant investigation.

12. Risks & Mitigations

High

AI Hallucination

AI may generate incorrect or fabricated root causes or workarounds. Mitigation: RAG with citations; require human verification for critical suggestions; feedback loop to improve retrieval; guardrails and output validation.

Medium

Azure OpenAI Provisioning

Delays in Azure OpenAI capacity or approval. Mitigation: Early procurement; fallback to alternative models if needed.

Medium

CA Tool API Stability

CA Tool APIs may change or have rate limits. Mitigation: Abstract behind Sprout DS; caching; monitor and version API contracts.

Low

Chrome Side Panel Adoption

Users may not discover or persist with sidebar. Mitigation: Onboarding prompts; default to open when on Jira; measure engagement.

14. Appendix / Glossary

RAG (Retrieval-Augmented Generation)
A technique that combines retrieval of relevant documents with LLM generation. The model is given retrieved context (e.g., similar tickets) to produce more accurate, grounded responses with citations.
Customer Advocacy (CA) Tool
Internal tool for tracking EAB (Escalation, Advocacy, Bypass) metrics. Used by Customer Advocacy Analysts and L4 engineers. Used for customer advocacy workflows.
EAB
Escalation, Advocacy, Bypass â€” key metrics tracked in the CA Tool for support operations.
L4 Support Engineer
Level 4 support engineer; handles complex escalations, root cause analysis, and customer advocacy cases.
Chrome Side Panel
Chrome MV3 API that allows extensions to render a persistent sidebar panel alongside web pages (e.g., Jira), as opposed to a popup.
